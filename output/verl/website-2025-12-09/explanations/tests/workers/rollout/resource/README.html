1. 📂 当前这个文件夹（resource）主要负责什么功能？—— “演习物资仓库”

如果说 tests/workers/rollout/ 里的代码是“演习剧本”（规定了怎么测试、测试什么流程），那么这个 resource 文件夹就是“演习物资仓库”。

  • 功能： 它存放了所有测试过程中需要的静态文件和配置文件。
  • 作用： 测试代码运行时，不能凭空变出数据或配置，它需要来这个仓库里“取货”。比如测试 AI 使用工具的能力时，代码会来这里读取“工具到底该怎么配”的信息。
  • 人话： 剧组拍戏（运行测试）之前，得先来道具间（Resource）领道具。

2. 📄 这个文件夹下的各个直接文件分别是干什么的？

👀 观察： 根据你提供的目录结构，tests/workers/rollout/resource/ 的根目录下没有任何直接文件（没有 .json 或 .py 散落在地上）。

  • 现状： 这个仓库的“地板”是干净的，没有散落的零件。
  • 含义： 这说明所有的资源都被分门别类地装进了子文件夹（箱子）里，管理非常井井有条。

3. 📁 子文件夹的作用是什么？

这里只有一个子文件夹（箱子）：

📁 tool_configs/ —— “特工装备说明书合集”

  • 概括： 这个箱子里装的都是给 AI 模型使用的外挂技能配置。
  • 作用： 里面定义了 AI 如何连接搜索工具、如何使用沙盒跑代码、以及相关的账号密码（Mock 数据）。它是告诉 AI “你的义肢该怎么用”的说明书集合。

4. 🧠 高层认知 —— “剧本 vs. 道具”

要理解为什么会有这个 resource 文件夹，你需要建立 “逻辑与数据分离” 的认知：

  • 外面的代码（Tests）： 是“动”的。它们像导演和演员，负责跑流程、断言结果、模拟交互。
  • 这里的资源（Resource）： 是“静”的。它们像道具和布景板。

一句话总结： 为了不让测试代码里写满乱七八糟的配置参数，我们将所有“演习”需要的“道具配置（如工具设置）”都打包扔进了这个 resource 仓库里，随用随取。
